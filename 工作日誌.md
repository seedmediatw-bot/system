## 2026-05-08

### HR 自動化系統（index.html + Cloudflare Worker）

#### 1. 報帳頁 ── 發票辨識功能（雙模式）
- 新增「收據 / 發票」區塊，支援兩種模式切換：
  - **一般模式**：即時拍照（`capture="environment"` 支援手機鏡頭）或從相簿選取，照片縮圖即時顯示於清單，可逐張刪除
  - **AI 辨識模式**：選取多張發票照片，送至 Gemini API 自動辨識，回傳 JSON 後自動填入費用明細（說明、類別、金額、日期）
- 辨識結果會疊加到已有明細，不會覆蓋手動輸入的項目

#### 2. 報帳頁 ── 收據圖片直傳 Notion
- 送出報帳時，一般模式的照片透過 Cloudflare Worker `/api/notion-upload` 端點上傳：
  1. `POST /v1/files` 向 Notion 申請上傳位置
  2. `PUT` 二進位資料到 Notion 簽署 URL
  3. 取得 `file_upload_id`，以 Image Block 附加到報帳頁面
- 圖片直接可在 Notion 頁面中預覽，不需外部連結

#### 3. 報帳送出 Bug 修正
- 移除 `buildExpenseProps` 中的 `收據連結` URL 欄位（該欄位不存在於 Notion DB，導致送出失敗）
- 補寫原本缺漏的兩個欄位：`員工編號`、`備註`（備註同時合併收據連結文字）

#### 4. Gemini API 金鑰安全性
- 移除前端 `localStorage` 儲存 Gemini API Key 的機制
- Gemini Key 改由 Cloudflare Worker 的環境變數（`GEMINI_API_KEY`）統一管理，前端不再暴露金鑰

#### 5. Cloudflare Worker 全面改版（user 主導）
- 使用者自行改寫 `cloudflare-worker.js`，以 Gemini API 取代 Claude API
- 合併最終版本：保留 `/api/notion-upload` 端點、新增 `/api/gemini` 支援動態 `model` 參數與 `x-gemini-key` header 備援、加回 404 fallback
- `/api/status` 改回傳 `gemini` 狀態欄位（取代 `claude`）
- CORS headers 新增 `PUT` method 與 `x-gemini-key`

#### 6. AI 助理改接 Gemini
- `sendAI()` 從呼叫 `/api/claude`（Claude Messages 格式）改為呼叫 `/api/gemini`（Gemini `contents/systemInstruction` 格式）
- 回應解析路徑更新為 `candidates[0].content.parts[0].text`

#### 7. Notion 報帳 DB 欄位比對
- 核查 Notion「💰 報帳申請」DB schema，確認所有程式碼寫入欄位均有對應
- 補寫缺漏欄位 `員工編號`（rich_text）與 `備註`（rich_text）

#### 3. 收據圖片自動上傳嘗試（進行中）
**嘗試方案 A：Notion Files API（失敗）**
- Step 1（申請上傳位置）成功，Step 2（上傳二進位）持續回 400
- Cloudflare Worker 多次調整上傳方式（PUT raw、POST FormData 至 `/send` 端點）均失敗
- 確認 Notion Files API 行為不穩定，暫時放棄此路線

**嘗試方案 B：Google Drive + Service Account（失敗）**
- 建立 Google Cloud Service Account，啟用 Google Drive API，建立共用資料夾
- 實作 Cloudflare Worker `/api/drive-upload` 端點（JWT 認證 → multipart 上傳 → 設定公開權限）
- 失敗原因：Google Service Account 無個人 Drive 儲存空間配額，無法上傳至一般 My Drive
- 解決需使用 Shared Drive（需 Google Workspace 付費版）
